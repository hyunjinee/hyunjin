## 3. 독립 블로그 글감

### A. 마이그레이션·버그 케이스스터디

1. **"React 19가 깨운 잠자던 레이스: 초당 1,300회 refetch 루프"** — React 18의 배칭이 유계로 눌러두던 인증 상태 진동(401→refresh 성공→재시도 401)이 React 19의 effect 스케줄링 변화로 발산, 계측 초당 약 1,300회. async 탈출 로직이 토글 속도를 못 따라간 구조까지 분석. '업그레이드가 버그를 만든 게 아니라 원래 있던 레이스를 드러냈다'는 서사. 마이그레이션 PR에서 테스트 2개를 it.skip 처리하고 티켓을 분리한 프로세스도 부제로 가능. (근거: BRZ-9596, BRZ-9450/9562, PR #7952)
2. **"라이브러리가 dist에 React 18을 몰래 번들했다"** — 사내 UI 라이브러리의 rollup external이 정확 문자열 매칭이라 react/jsx-runtime 서브패스가 externalize되지 않아 152곳의 import를 따라 react@18 전체가 번들, React 19의 내부 API 제거로 SSG 크래시. 수정은 external 정규식화 + peerDependencies. '라이브러리 패키징의 정석'을 실제 사고로 설명. (근거: BRZ-9584)
3. **"Node 21의 navigator 전역이 SSG를 깨뜨렸다"** — Node 20 EOL(2026-04-30)로 24 직행 중, navigator 전역 신설로 lottie 라이브러리가 prerender에서 document 접근 크래시. 프로덕션 코드 무수정으로 서버 빌드에서만 no-op stub 치환. 검증 약 3,500개 테스트 그린. (근거: BRZ-9374, PR #7899)
4. **"정적 HTML에 에러 화면이 구워졌다"** — 동기 UUID 가드가 빌드 prerender의 placeholder(_postId)를 잘못된 UUID로 판정해 정적 HTML에 ErrorUI가 박히고, 푸시 랜딩 시 한 프레임 노출 후 하이드레이션으로 교체되는 플리커. placeholder만 가드 우회, 런타임 보호 유지. SSG+CSR 하이브리드의 함정. (근거: 커밋 aa550eabb + BRZ-7177/#7026, 같은 사건 병합)
5. **"클라이언트 라우터는 /ko.txt를 찾는데 S3에는 /ko/index.txt만 있다"** — Next.js 클라이언트 라우터의 RSC payload 요청 형태와 빌드 산출물 형태의 불일치가 CDN 함수의 catch-all 패턴에 잘못 매칭돼 하루 약 20건 오호출. 진입 시 경로 정규화로 해결. 'static export + CDN에서 RSC payload 라우팅은 직접 책임져야 한다'는 교훈. (근거: BRZ-5416, #6436)
6. **"ESLint 8에서 10으로 직행하기"** — 블로커였던 archived 프리셋(@vercel/style-guide)을 버리는 결정, typescript-eslint만 먼저 올리는 PR과 flat 전환 PR의 2-PR 위험 격리, turbo 캐시 무효화 디테일. (근거: BRZ-9438, #7930)
